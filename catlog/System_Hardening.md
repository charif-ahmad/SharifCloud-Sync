# 🛡️ SharifCloud-Sync | Security Architecture Report

## 1. ملخص التحصين (Security Hardening Summary)
تم نقل المشروع من بيئة تطوير مفتوحة (Development) إلى بيئة إنتاج معزولة (Isolated Production) تعتمد على مبدأ **"Zero Trust"** داخل شبكة خاصة.

---

## 2. التغييرات التقنية والأسباب (Technical Changelog)

| المكون | التغيير التقني | لماذا؟ (السبب الأمني) |
| :--- | :--- | :--- |
| **الشبكة (Networking)** | الاعتماد على **Tailscale VPN** بدلاً من الـ IP المحلي أو فتح البوروتات. | منع هجمات الـ Public Internet وتشفير البيانات بين الموبايل والسيرفر بتشفير (WireGuard). |
| **Web Server (Nginx)** | حصر الاستماع (`listen`) على IP التايلسكيل `100.x.x.x` فقط. | عزل الواجهة الأمامية عن أي متصل بشبكة الواي فاي المنزلية (LAN Isolation). |
| **API (Node.js)** | تقييد `app.listen` بـ IP التايلسكيل ومنع الـ `0.0.0.0`. | منع الوصول المباشر لبورت 3000 من داخل الشبكة المحلية وتجاوز Nginx. |
| **المصادقة (Auth)** | استخدام **X-API-Key** داخل الـ Headers وليس الـ URL. | منع تسريب المفتاح في سجلات المتصفح (Browser History) أو الـ Access Logs. |
| **الحماية (Middleware)** | إضافة **Helmet.js** و **Rate Limiting**. | حماية الـ Headers من الهجمات الشائعة ومنع هجمات التخمين (Brute Force) والـ DoS. |

---

## 3. رحلة الطلب المؤمن (Secure Request Flow)

1.  **الموبايل:** يرسل طلب عبر تطبيق Tailscale (نفق مشفر).
2.  **الراوتر:** يمر الطلب من خلاله كبيانات مشفرة لا يفهمها.
3.  **Nginx:** يستقبل الطلب لأنه قادم من IP موثوق، ويحول الطلب للباك إند.
4.  **Node.js:** يتحقق من الـ `X-API-Key` ويرد بالبيانات.
5.  **النتيجة:** أمان بنسبة 100% ضد المتطفلين الخارجيين والداخليين.
